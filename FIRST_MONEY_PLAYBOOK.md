# 第一笔钱 48 小时执行剧本

目标：用一个范围很小的服务拿到第一笔合法收入。不要先做大产品，不要等账号、品牌、作品集都完美。

## 推荐卖什么

卖“24小时小网站 / GitHub Pages 页面 / 网站小修”。

起步报价：

- 单页网站或 link-in-bio：49 美元
- 小商家活动页、菜单页、课程页：99 美元
- 网站小修、表单整理、轻量自动化：79 美元

为什么选这个：

- Codex 可以帮你快速做页面、修代码、写文案。
- GitHub 可以当作品证明和部署渠道。
- 客户能理解价值，不需要解释复杂技术。
- 第一单可以小而快，不需要长期信任。

## 今天要做的 6 件事

1. 把 `index.html` 里的邮箱、GitHub、名字改成你的真实信息。
2. 新建一个 GitHub repo，例如 `fast-web-help`。
3. 上传 `index.html`，开启 GitHub Pages。
4. 把页面链接发给 30 个潜在客户。
5. 每个客户只推一个小结果：页面、修复或自动化，不推“全能开发”。
6. 对第一个回复的人给出固定小报价和明确交付时间。

## 最容易成交的客户

- Instagram、TikTok、X、微信、小红书上有内容但没有独立页面的创作者。
- Google Maps/Yelp 上页面很旧的小商家。
- 正在卖课、活动、咨询、作品集但只用聊天软件收集信息的人。
- GitHub 上有项目但 README 和 demo 很弱的独立开发者。
- 本地服务人员：教练、摄影、维修、翻译、简历修改、家教。

## 第一封消息的规则

- 不要群发长文。
- 不要说“我可以做任何网站”。
- 先指出一个具体可改善点。
- 给一个很小、低风险、固定价格的结果。

## 付款建议

第一单尽量使用客户熟悉且你能收款的方式。常见选择包括 PayPal、Stripe Payment Link、Wise、微信/支付宝或平台托管。没有信任基础时，建议先收 30%-50% 订金，或通过 Upwork/Fiverr 这类平台托管。

## 平台优先级

主动外联优先于等平台流量：

1. 直接找 30 个目标客户，发短消息。
2. 同步发布 Fiverr/Upwork 服务，但不要等待。
3. 把 GitHub Pages 链接当作作品页。
4. 拿到第一单后，马上请求一句评价和允许展示的截图。

## 已核对的现实约束

- GitHub Pages 支持用仓库里的 `index.html` 作为站点入口，适合这个静态服务页。
- Upwork 的自由职业者服务费会按合同浮动，官方说明范围是 0%-15%，报价时要把平台费用算进去。
- Fiverr 的官方帮助说明里，订单完成后的收入通常要等待结算期；所以它适合挂服务，但不应作为第一天唯一获客渠道。

## 成交话术

当对方有兴趣时：

> 我可以先做一个 24 小时内可上线的第一版，包含移动端适配、联系按钮和一次小修改。这个范围我按 $49 做。你只需要给我文字、Logo/图片和联系方式。可以的话我今天开始，明天给你链接。

如果对方犹豫：

> 没关系。我可以先免费发你一个 3 点改进建议，你觉得方向对，再决定要不要做。

## 今天的成功标准

不是“赚很多钱”，而是完成这三个动作：

- 有一个可发的作品页链接。
- 发出 30 条个性化外联。
- 至少拿到 3 个回复或 1 个明确报价谈判。
